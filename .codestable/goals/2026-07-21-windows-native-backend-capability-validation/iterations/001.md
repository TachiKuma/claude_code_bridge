# Iteration 001

## Current Understanding

本轮目标是验证候选 Windows native backend 基座能否胜任 `ccb` 在 Windows 上的完整运行需求，而不是只重复上一轮 gate。已知起点是：旧 gate 工作验收通过，但业务结论未通过；`rmux 0.9.0` 卡在 `start-server`，`psmux 3.3.3` 有 6 个 required gaps。

## Implementation Approach

先做低风险 fresh evidence：版本、安装来源、当前版本 full probe、gap-specific 手动黑盒验证和 `rmux` 启动形态诊断。对需要改变本机工具状态的 `psmux 3.3.7` 安装/替换，按项目危险操作规则进入 owner-stop，不静默执行。

## Changes This Iteration

- 新建 goal 起点报告和 `state.yaml`。
- 运行当前 `psmux 3.3.3` fresh capability probe，仍为 6 个 blocking gaps。
- 运行当前 `rmux 0.9.0` fresh capability probe，默认、`RMUX_DISABLE_TINY_CLI=1`、补完整安装目录 PATH 后均仍为 1 个 blocking gap。
- 增加 `evidence/gap-specific-summary.json`，记录 6 个 `psmux` gap 的逐项复核结论。
- 增加 `evidence/rmux-diagnostics-summary.json`，记录 `rmux start-server` invocation-shape 敏感问题。
- 新增 `approval-report.md`，请求是否允许安装/切换到 `psmux 3.3.7` 后重跑 gate。

## Verification Evidence

- `python scripts/probe_rmux_capability.py --work-root .../evidence/psmux-current --rmux-bin psmux`：completed，blocking gaps = 6。
- `python scripts/probe_rmux_capability.py --work-root .../evidence/rmux-current --rmux-bin rmux`：completed，blocking gaps = 1。
- `RMUX_DISABLE_TINY_CLI=1` 后重跑 `rmux` probe：completed，blocking gaps = 1。
- 补 `rmux` 完整安装目录和 `libexec/rmux` 到 PATH 后重跑：仍然 blocking gaps = 1。
- `psmux list-commands`：没有 `set-window-option`，但有 `set-option`、`show-options`、`show-window-options`、`select-pane`。
- 手动 psmux pane option 复核：`set-option -p` 返回 0，但 `#{@ccb_slot}` / `show-options -p` 读回为空；`select-pane -T` 可读回 title。
- 手动 psmux paste 复核：`load-buffer` + `paste-buffer` + 显式 `Enter` 后 capture 可看到 marker。
- 手动 psmux control-key 复核：`C-c` 可中断 Python 并出现 `KeyboardInterrupt`；`C-d` 只显示为 `^D`，不触发 EOF。
- GitHub API / Winget：`psmux` 最新 `v3.3.7`，本机 PATH 为 `psmux 3.3.3`；`rmux` 最新为 `v0.9.0`。
- 进程检查：本轮临时 `rmux` daemon 已按 namespace 清理，未留下 probe namespace 残留。

## Problems Encountered

- 当前 `psmux 3.3.3` 仍不能通过完整 gate。部分 gap 可降级或修正 probe 语义，但 pane user option 和 interactive attach 仍是完整 `ccb` lifecycle 的实质阻断。
- `rmux 0.9.0 start-server` 对 stdout/stderr capture 敏感：不 capture 时能很快返回，capture 时超时。这解释了 probe 失败，也说明当前 `PsmuxBackend` 继承的 capture 调用形态不能直接用于 `rmux start-server`。
- 验证 `psmux 3.3.7` 需要安装或替换本机可执行文件，属于包管理/执行外部可执行文件变更，需要 owner 明确确认。
