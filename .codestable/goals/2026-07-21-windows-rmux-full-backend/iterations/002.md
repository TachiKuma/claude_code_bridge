# Iteration 002

## Current Understanding

首次 Acceptance agent 验收结论为 fail。通过项包括 stdio-aware lifecycle、logical EOF、capture tail/parser fidelity 和 full gate；失败点集中在真实 CCB 集成：

- `start_foreground` 的 rmux attach 分支依赖 ping payload 中的 `namespace_backend_impl=rmux` 等字段。
- `ProjectNamespaceState.summary_fields()` 尚未输出 rmux canonical namespace metadata。
- `ProjectNamespaceController` 默认 backend factory 仍固定到 `TmuxBackend`，未受 `CCB_TERMINAL_BACKEND=rmux` 驱动。
- native Windows rmux namespace bootstrap 不能继续使用 Unix `sh -lc while ...` placeholder。

## Implementation Approach

本轮采用 additive 兼容修复：

- legacy `tmux_socket_path` / `tmux_session_name` 字段保留，避免破坏现有 tmux 调用方。
- 新增 canonical summary 字段：`namespace_backend_impl`、`namespace_id`、`namespace_session_name`、`namespace_ipc_kind`、`namespace_ipc_ref`。
- controller 默认 backend factory 复用现有 `terminal_runtime.get_backend_for_session()`，由 `CCB_TERMINAL_BACKEND` / `CCB_MUX_BACKEND` 选择 rmux，不新增配置格式。
- `build_backend()` 支持 `namespace=`，tmux/fake backend 不支持时回退旧 `socket_path=` 调用。
- placeholder 按平台返回，Windows 使用 PowerShell 长睡眠，非 Windows 保持 `sh -lc`。

## Changes This Iteration

- 更新 `ProjectNamespaceState`：
  - 增加 backend/ipc optional 字段；
  - `to_record()` 保存 raw optional 字段，保持 round-trip；
  - `summary_fields()` 输出 resolved canonical namespace metadata。
- 更新 `ProjectNamespace` model 和 create/reflow 返回路径，暴露 resolved backend/ipc metadata。
- 更新 namespace backend 构造路径：
  - `ProjectNamespaceController` 默认使用 `default_project_namespace_backend()`；
  - env 指定 `rmux` 时传入 `namespace_id` / `rmux_namespace`；
  - ensure/recreate/destroy/reflow/cmd-slot rebuild backend 时传入 project namespace。
- 更新 `terminal_runtime.placeholders`：
  - Windows `pane_placeholder_argv()` 返回 `powershell.exe -NoProfile -Command "while ($true) { Start-Sleep -Seconds 3600 }"`；
  - 非 Windows 行为不变。
- 补测试：
  - rmux namespace summary metadata；
  - controller 默认 factory 使用 `CCB_TERMINAL_BACKEND=rmux`；
  - ping payload 透传 rmux namespace attach metadata；
  - create session 测试适配平台化 placeholder 和新增 `has-session` 观测。
