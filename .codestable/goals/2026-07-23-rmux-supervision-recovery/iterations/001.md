# Iteration 001

## Current Understanding

本 goal 的核心风险是把 pane、process/job、namespace、daemon 四类 evidence 混成单一 health。第一轮先建立 supervision 层的 evidence ledger 和 backend-neutral matching，不改 provider parser、backend resolver、namespace lifecycle 或 process liveness implementation。

## Implementation Approach

- 新增 supervision-owned evidence ledger，复用 `AgentRuntime` 既有 `namespace_ref`、`pane_ref`、`process_ref` 字段。
- 将 Rmux pane id 识别从 `%N` 前缀解耦，允许 backend-local pane id。
- 将 project namespace 归属判断从 tmux socket 扩展到 canonical namespace session/id。
- 扩展 recoverable health set，支持 process/job death 与 namespace crash。
- recovery event 在存在 explicit evidence 时写入 `evidence_ledger`。
