# Iteration 001

## Current Understanding

`mux-backend-contract` 的实现边界是新增 backend-neutral mux contract、fake backend 测试替身和 `_tmux_run` 泄漏点 inventory；不迁移 provider session payload、不修改 ccbd control-plane transport，也不实现 RmuxBackend。

方案深度选择沿用已 approved design：新增 capability-specific protocols 和组合 facade。fake backend 是长期测试资产，替代外部 mux 进程边界，但必须真实建模 namespace/window/pane 状态，不用 tmux argv recorder。

## Implementation Approach

- 新增 `lib/terminal_runtime/mux_backend_contract.py`，定义 refs、capabilities、structured error、小协议和组合 `MuxBackend` facade。
- 新增 `lib/terminal_runtime/fake_mux_backend.py`，提供状态机式 fake backend、event log 和 failure injection。
- 新增 `test/test_mux_backend_contract.py` 覆盖 import contract、协议拆分、backend-neutral refs、fake state/event、structured error、semantic split direction 和旧 `TmuxBackend` public methods。
- 新增 `.codestable/features/2026-07-19-mux-backend-contract/mux-backend-contract-leakage-inventory.md` 记录迁移输入。
