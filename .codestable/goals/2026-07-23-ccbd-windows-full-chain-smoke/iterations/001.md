# Iteration 001

## Current Understanding

本 goal 的最小安全推进点是先交付可复核的验证资产，而不是直接执行 native Windows 真机 start/ask/kill。真实 CMD-004 会启动项目 runtime、提交 `ccb ask`，并执行 `ccb kill -f` 清理本地资源，属于需要 owner 风险接受的动作。

## Implementation Approach

本轮采用验证资产与生产 runtime 分离的方案：

- Python parser 只做 transcript 判定与 scope guard，不执行 runtime 命令。
- PowerShell runner 只负责 native Windows 证据采集、redaction、transcript 生成和 finally cleanup。
- 测试覆盖 fail-closed 负例，不使用 fake backend 或 probe 输出作为 pass。
