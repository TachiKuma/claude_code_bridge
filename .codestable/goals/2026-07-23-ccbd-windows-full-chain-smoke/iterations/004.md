# Iteration 004

## Current Understanding

Iteration 003 的真实 PS5/PS7 smoke 已经通过，但独立代码审查发现四个必须补齐的问题：rmux 本地 pane id 仍受 tmux `%` 假设影响、warm reuse 仍只接受 `tmux:`、scope guard 未对未知路径 fail-closed、redaction 漏掉 `access_token` / `refresh_token`。

## Implementation Approach

本轮只修这些审查项，不扩大到 packaging/docs、provider parser 或完整 validation matrix。运行时修复集中在 mux runtime ref / pane id 解析，parser 修复集中在 fail-closed scope guard 和 token redaction。

## Changes This Iteration

- `binding_runtime.common` 增加 mux runtime ref 解析，支持 `tmux:`、`rmux:`、`psmux:`；tmux 仍要求 `%` pane id，rmux/psmux 接受 backend-local pane id。
- binding validation 和 start preparation 从 tmux-only 判定改为 mux 判定，`rmux:pane-C` 可解析为 pane id 并进入 warm reuse 检查。
- `start_agent_runtime` 在 `namespace_backend_impl=rmux` 时接受 `assigned_pane_id='pane-C'`，不再要求 tmux socket path 才记录 namespace runtime evidence。
- `agent_runtime_binding.runtime_pane_facts` 使用 mux runtime ref parser，避免 rmux 复用路径丢失 pane fact。
- smoke parser scope guard 改为白名单 fail-closed：未知路径默认拒绝，仅允许本 goal / issue / artifact 前缀和明确列出的实现、测试、脚本文件。
- Python 与 PowerShell redaction 同步支持 `access_token`、`refresh_token` 等 `*_token` key，同时保持 `stage=ask-evidence` 不误伤。
- 增加测试覆盖 rmux local pane id、rmux warm reuse、unknown path fail-closed、token redaction。
- 修正一个 Windows TOML fixture：测试配置中的临时路径写为正斜杠，避免 `\U` 被 TOML 解析为非法 Unicode escape。
