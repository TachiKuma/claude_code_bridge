# Iteration 001

## Current Understanding

上一阶段完成的是仓库内 backend 接入切片，后续必须先验证 Windows mux 能力。`docs/ccbd-windows-psmux-plan.md` 明确：没有 capability report，不应进入 implementation phase。

## Implementation Approach

采用真实 probe，不使用 mock。先运行默认 `rmux`，再运行本机存在的 `psmux`，以区分 `rmux` 包装器入口问题和 `psmux` 实际能力缺口。发现 probe 脚本在 Windows 下遇到卡死命令时不会可靠完成报告，因此先修 probe 工具，再重新采集最终证据。

## Changes This Iteration

- `scripts/probe_rmux_capability.py` 的 `SubprocessRunner` 从 `subprocess.run()` 改为 `Popen.communicate(timeout=...)`，Windows timeout 时用 `taskkill /T /F` 清理子进程树。
- 核心命令 `start-server` / `new-session` / `has-session` unsupported 时，probe 现在会 early-stop，写出 `capability-report.json`，并记录 blocking gap。
- early-stop 路径会按 probe namespace 清理 orphan `rmux.exe` / `psmux.exe` daemon，并写 `artifacts/cleanup/orphan-daemons.json`。
- report 的 `backend_impl` 从实际二进制名派生，避免 `--rmux-bin psmux` 时误标成 `rmux`。
- 新增 probe 单测覆盖 core command early-stop report 和 backend_impl 派生。

## Capability Gate Result

`rmux 0.9.0`：

- `probe_status`: completed。
- blocking gap：`command:start-server` unsupported，returncode 124，degrade impact `core-lifecycle`，consequence `无法建立 mux server`。
- 结论：默认 `rmux` 路径不能作为 Windows native backend lifecycle 基座。

`psmux 3.3.3`：

- `probe_status`: completed。
- supported commands 包括 `start-server`、`new-session`、`has-session`、`list-windows`、`split-window`、`list-panes`、`send-keys`、`capture-pane`、`pipe-pane`、`respawn-pane`、`kill-session`、`kill-server` 等。
- supported semantics 包括 `session_survival`、`namespace_isolation`、`window_policy`、`layout_reflow`、`pane_id_stability`、`capture_last_n_lines`、`pane_death`、`kill_session_cleanup` 等。
- required blocking gaps：
  - `command:set-window-option` unsupported，无法写 window option evidence。
  - `semantic:attach_reattach` partial，用户无法恢复 UI。
  - `semantic:user_options_title` partial，pane identity marker 不可靠。
  - `semantic:capture_format_fidelity_for_provider_completion` partial，provider pane status 可能静默误判。
  - `semantic:buffer_paste` partial，大文本输入不可用。
  - `semantic:ctrl_c_ctrl_d` partial，interrupt / EOF 语义不可控。
- 结论：`psmux` 比 `rmux` 更接近可用，但 required gap 未清零，不能进入完整 `ccbd` namespace lifecycle 实装。
