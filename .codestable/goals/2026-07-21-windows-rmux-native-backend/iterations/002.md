# Iteration 002

## Current Understanding

Task agent 对 iteration 001 的只读验收给出 fail，主因是 `PsmuxBackend` 虽然不在命令行上传 tmux `-f` config 参数，但复用了 `TmuxBackend._tmux_run()` 的环境隔离路径，可能把 `CCB_TMUX_CONFIG` 传给 rmux 子进程。该问题直接触碰“不泄漏 tmux config”的 acceptance，需要修复后复验。

## Implementation Approach

保留 tmux 默认路径不变，在 `TmuxBackend` 中提取 `_command_env()` 扩展点；`PsmuxBackend` 覆盖该方法并移除 `CCB_TMUX_CONFIG`。这比改全局 `isolated_tmux_env()` 更保守，避免影响现有 tmux 行为。

## Changes This Iteration

- `TmuxBackend._tmux_run()` 改为调用 `self._command_env()`。
- `TmuxBackend._command_env()` 保持原有 tmux 隔离环境。
- `PsmuxBackend._command_env()` 在父类隔离基础上移除 `CCB_TMUX_CONFIG`。
- `test_psmux_backend_uses_rmux_namespace_without_tmux_config` 增加子进程 env 断言。
- 新增 `test_backend_selection_uses_backend_impl_and_mux_backend_fields`，覆盖 `backend_impl` 与 `mux_backend` 字段选择。
