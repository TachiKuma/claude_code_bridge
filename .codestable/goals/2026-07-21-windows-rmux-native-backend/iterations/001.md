# Iteration 001

## Current Understanding

`windows-rmux-native-backend` 的当前权威路线是 `docs/ccbd-windows-psmux-plan.md`，不是旧 WezTerm 路线。该文档要求先完成 capability gate 和 mux backend contract 收口；仓库已有 rmux capability probe，但 runtime 层还没有可选择的 psmux/rmux backend。

## Implementation Approach

方案深度选择为“真实 runtime 接入切片”：新增 `PsmuxBackend` 并复用 tmux-family 命令语义，扩展显式 backend 选择。没有选择 fake/stub，因为 backend 类会成为长期入口；没有直接做完整 ccbd lifecycle，因为缺少 Windows 真机 capability report，贸然接入会越过设计闸门。

## Changes This Iteration

- 新增 `lib/terminal_runtime/psmux_backend.py`，提供 `PsmuxBackend`。
- 为 `TmuxBackend` 与 `PsmuxBackend` 增加 `backend_family` / `backend_impl` 标识。
- 扩展 `TerminalBackendSelection`，支持 `terminal_backend`、`backend_impl`、`mux_backend`、`psmux` 和 `rmux` 显式选择。
- 让 `terminal_runtime.api` 与包入口导出 `PsmuxBackend`，并支持 `CCB_TERMINAL_BACKEND` / `CCB_MUX_BACKEND` 显式选择。
- 抽出 tmux-family 命令 base 构造，确保 psmux/rmux 使用 `rmux -L namespace` 且不继承 `CCB_TMUX_CONFIG`。
- 修正 backend 路径参数处理：仅展开 `~`，不在 Windows 上把 POSIX 风格 socket/config 字符串改写为反斜杠。
- 新增 backend 选择和 psmux 命令构造测试。
