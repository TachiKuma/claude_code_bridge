# Iteration 001

## Current Understanding

本 feature 的关键边界是 evidence-first：`ccbd` 仍是 runtime authority，Job Object / process tree / pane 只提供 evidence。实现必须集中在 `provider_runtime/process_ref.py` seam，不能让 health、kill、helper cleanup 各自解释 `kind/evidence_state/runtime_generation`。

## Implementation Approach

采用最小契约扩展：新增 canonical `process_ref` builder / normalizer / cleanup eligibility seam，并把现有 runtime record、provider facts、helper manifest、health、kill cleanup、project view diagnostics 接到该 seam。没有引入真实 Windows Job Object handle 持久化，也没有改变 `ccb kill` 的用户语义。

## Changes This Iteration

- 新增 `lib/provider_runtime/process_ref.py`，提供 `ProcessRef`、`build_process_ref()`、`process_ref_from_record()`、`process_ref_to_record()`、`process_ref_health()` 和 `process_ref_allows_destructive_cleanup()`。
- `AgentRuntime` / `AgentRuntimeStore` 增加 `process_ref` roundtrip，并通过 normalizer 剔除非 canonical 字段。
- `ProviderRuntimeFacts` 与 health rebind path 增加 `process_ref` 派生和回写。
- `ProviderHelperManifest` 增加 helper-specific `process_ref`，避免复用不匹配 bridge leader 的 runtime process evidence。
- `runtime_health()` 在 pane health 前消费 process evidence；missing / stale / degraded / unsupported process evidence 会让 runtime 进入 degraded health。
- `runtime_pid_cleanup` 收集 process_ref pid，并把 Windows `pid_matches_project()` 从无条件 `True` 改为 process_ref / authority-record / explicit control-plane evidence。
- `project_view` agent record 增加 `process_ref` 诊断投影。
- `rust_helpers` 在 Windows 下用当前 Python 解释器运行 `.py` helper，修复 project view DoD helper 用例。
- checklist 全部写回 `done`，roadmap item 写回 `done`。

## Problems Encountered

独立 review 发现并已修复 4 项 blocking：

- runtime record 曾原样持久化未规范化的 `process_ref`，可能把 handle 字段写入记录。
- helper manifest 曾复用 runtime `process_ref`，当 runtime pid 与 bridge leader pid 不一致时会导致 Windows helper cleanup 自拒绝。
- destructive cleanup eligibility 曾未强制 runtime_generation。
- Windows ccbd lease / keeper authority pid 曾缺少无需 `/proc` / `ps` 的明确 authority-record 放行路径。

此外，project view DoD 中的 `.py` helper 在 Windows 下不能依赖 POSIX shebang 执行，已在 helper runner 中补 Windows `.py` 执行路径。
