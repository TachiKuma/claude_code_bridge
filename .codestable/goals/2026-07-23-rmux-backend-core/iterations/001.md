# Iteration 001

## Current Understanding

本 goal 目标是完成 `rmux-backend-core`：在 `MuxBackend` 契约下实现独立 `RmuxBackend` core adapter，覆盖 capability guard、command client seam、namespace/window、pane、presentation 和 error evidence。

关键边界是 core-only：不做 send/capture/logging、provider parser、foreground attach 或 ccbd lifecycle 接入。

## Implementation Approach

采用 production core adapter，而不是继续沿用旧 `PsmuxBackend` / `TmuxBackend` shim。

- `rmux_backend.py` 作为 public entry。
- `rmux_backend_runtime/` 分层承载 capabilities、client、errors、namespace、panes、presentation。
- 测试使用 fake Rmux command client，避免依赖真实 daemon。
- capability report 缺失或 required command unsupported 时 fail-fast。
