# Iteration 001

## Current Understanding

本 goal 的核心风险不是让 fake/subset 变绿，而是避免 WSL、probe、fake lane 或直接 rmux 诊断被误记为 native Windows true-host evidence。因此实现重点放在 manifest 分层、transcript fail-closed、failure classification、runbook schema 和 scope guard。

## Implementation Approach

- 新建专项 matrix/report 脚本，不扩写既有 phase6 或 multi-workgroup smoke，保持职责单一。
- 将 fake、provider_blackbox、windows_true_host、manual_transcript 分成明确 lane，并同时输出 `selected_cases_status` 与 `full_matrix_status`。
- PowerShell runbook 产出可解析 transcript sidecar，Markdown runbook 说明同一 schema。
- direct rmux 只允许命名诊断；其余直接 rmux/psmux 命令均判 `test_design_failure`。
- 通过 scope guard 约束本 feature 不修改 packaging/docs contract、backend implementation 或 provider parser。
