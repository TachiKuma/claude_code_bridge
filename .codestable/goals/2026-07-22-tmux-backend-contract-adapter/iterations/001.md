# Iteration 001

## Current Understanding

本 goal 的核心不是删除 tmux 内部 `_tmux_run`，而是把生产 `TmuxBackend` 包装成 `MuxBackend` 语义 adapter，并迁移代表性上层 seam。方案深度选择真实 adapter：`_tmux_run` 仅作为 adapter 私有实现，fake 只用于测试外部 tmux CLI 边界。

## Implementation Approach

新增 `TmuxMuxBackendAdapter` 包装现有 `TmuxBackend`，保持旧 public methods 不动。调用方迁移采用能力检测：传入 mux backend 时走 `namespace_ref()`、`create_session()`、`reflow_window()`、`swap_pane()` 等语义方法；传入旧 `TmuxBackend` 或既有测试 fake 时保留 `_tmux_run` fallback。

## Changes This Iteration

- 新增 `lib/terminal_runtime/tmux_mux_backend.py`，实现 refs、capabilities、namespace lifecycle、window layout/reflow、pane io/presentation/logging、diagnostics 和 tmux-to-mux error mapping。
- 导出 `TmuxMuxBackendAdapter` 到 `terminal_runtime.api` 与包入口。
- `layouts_root.py` 支持 mux backend detached root pane 路径，并移除 `TmuxLayoutBackend` protocol 对 `_tmux_run` 的 public 要求。
- `ccbd/services/project_namespace_runtime/backend.py` 的代表性 namespace/window helper 支持 mux backend 语义路径。
- `cli/services/runtime_launch_runtime/tmux_panes.py` 的 detached server policy 与 detached pane creation 支持 mux backend 语义路径。
- `agent_window_reflow.py` 支持无 `_tmux_run` 的 mux reflow ops：pane 几何观测、layout 应用和 swap 分别经 `describe_window_panes()`、`reflow_window()`、`swap_pane()`。
- 新增 `test/test_tmux_mux_backend_adapter.py`，并扩展 `test_agent_window_reflow.py`。
- 回写 feature checklist 为 `done`，新增 `tmux-backend-contract-adapter-leakage-inventory.md`。
