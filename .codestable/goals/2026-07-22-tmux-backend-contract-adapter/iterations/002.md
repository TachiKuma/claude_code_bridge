# Iteration 002

## Current Understanding

独立 review agent `019f89d0-fa32-7522-a3aa-2a9cdc58a080` 对 iteration 001 给出 `changes_requested`。主要问题是 mux 路径下 bare session target、错误分类/evidence、teardown、默认生产接入和 detached policy best-effort 语义不完整。

## Implementation Approach

保持原 adapter 方案，不扩大范围。修复集中在语义缺口：adapter 补旧 `TerminalBackend` facade 以允许默认 tmux backend 包装；ccbd helper 对 mux backend 走正确 bare-session/kill_server 方法；错误映射补齐 absent/missing/`subprocess.cmd` evidence；tmux server policy 常量抽到共享模块，避免复制 shell policy。

## Changes This Iteration

- `TmuxMuxBackendAdapter` 补 `is_alive()`、`activate()`、`create_pane()`、`get_current_pane_id()`、`get_text()`、`kill_tmux_pane()`、pane title/user-option/style 等旧 facade。
- `split_pane()` 同时支持 mux `MuxPaneRef` 和旧字符串 pane id，避免默认 adapter 接入破坏旧 layout caller。
- `attach_namespace()` 保持前台 attach `capture=False`。
- `session_root_pane()` 在 ccbd mux path 下直接调用 `backend.session_root_pane(namespace)`，不再经 `session:session` target。
- `kill_server()` 在 ccbd mux path 下调用 `backend.kill_server()`。
- `get_backend()`、`get_backend_for_session()` 和 project namespace 默认 tmux factory 包装 `TmuxBackend` 为 `TmuxMuxBackendAdapter`；`create_auto_layout()` 仍保留旧 `TmuxBackend` factory 以降低旧 layout 回归面。
- `_error_category()` 补 absent socket/server、`can't find window`；`_error_evidence()` 补 `TimeoutExpired` / `CalledProcessError` 的 `cmd` command evidence。
- 新增 `terminal_runtime/tmux_server_policy.py`，复用 clipboard/update-environment policy，避免 adapter 复制 shell 分支。
- `runtime_launch_runtime/tmux_panes.py` 的 mux detached policy 保持 best-effort，失败不阻断 pane launch。
- 扩展 `test/test_tmux_mux_backend_adapter.py` 覆盖 review findings。
