# Iteration 001

## Current Understanding

本轮目标是把 ccbd namespace state、ping/doctor payload 和 foreground attach 输入统一到 mux-neutral canonical schema，并保留旧 tmux alias 兼容。实现已落地，且上一轮 review 指出的覆盖风险已经修正。

## Implementation Approach

采用 canonical-first、alias-fallback 的收敛方式，只在 schema、payload 投影和输入解析层改动，不扩展 RmuxBackend 运行时能力，不改变 tmux attach 行为。

## Changes This Iteration

- `ProjectNamespaceState` / `ProjectNamespaceEvent` 增加 canonical namespace fields，并将 `namespace_id` 固定为 `project_id`，`namespace_backend_family` 固定为 `tmux-family`。
- `build_ccbd_payload()` 与 doctor summary 改为避免 plain `tmux_socket_path` / `tmux_session_name` 覆盖顶层 ccbd placement。
- `attach_started_project_namespace()` 改为 canonical-first，legacy fallback 保留。
- 增加 state、payload、render、attach 的回归测试，并回写 feature checklist / review / QA / roadmap。
