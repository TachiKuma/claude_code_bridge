# Iteration 001

## Current Understanding

本 goal 目标是完成 `provider-runtime-backend-session-contract`：把 provider launch、session payload、runtime health 和 provider env 迁移到 backend-neutral mux 字段，同时保留旧 tmux alias。

依赖 `mux-backend-contract`、`windows-namespace-ipc-schema` 和 `windows-job-object-runtime-evidence` 已完成；本轮继续完成实现、review、QA、功能验收和 roadmap writeback。

## Implementation Approach

采用持久契约级实现，不使用 fake / stub 替代核心逻辑。

- 新增共享 `provider_runtime.session_payload` helper，统一 canonical mux payload、protected merge、canonical-first reader/env projection 和 rebind writeback。
- provider launcher 只写 provider-specific payload，不再各自写 shared canonical tmux authority。
- reader / runtime facts / attach 持久化链路通过 canonical view 派生 backend-neutral 字段。
- provider env loader 保持 env 显式值优先，并从 session file 补齐缺失 canonical mux fields。

## Changes This Iteration

- 新增 `lib/provider_runtime/session_payload.py`。
- 更新 shared session writer，输出 `terminal=mux` 与 `backend_*`、`pane_ref`、`namespace_ref`、`compat`。
- 清理 Codex、Claude、Gemini、OpenCode、native CLI、Kimi、MiMo、DeepSeek、AGY、Droid provider launcher 的 canonical tmux payload 写法。
- 更新 pane log session reader、Claude session reader、session binding evidence、`ProviderRuntimeFacts` 与 runtime attach/refresh 持久化字段。
- 更新 Codex/Gemini/OpenCode env loader，`CCB_MUX_*` canonical-first，旧 `*_TMUX_SESSION` fallback。
- 修复 review 发现的三个 blocking：env session-file merge、`terminal=mux` recovery、rebind canonical `pane_ref` writeback。
- 补充 writer、reader、runtime facts、env loader、ensure_pane 和 guard 测试。
- 写入 feature review、QA、acceptance 与 goal functional acceptance 报告。
