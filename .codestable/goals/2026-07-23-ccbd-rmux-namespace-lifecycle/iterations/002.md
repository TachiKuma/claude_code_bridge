# Iteration 002

## Current Understanding

本轮继续处理 cross-cutting namespace readers。核心边界是：Rmux namespace 不应通过 legacy `tmux_socket_path` 被误投射成 tmux authority；但尚未设计 provider/runtime 层面的 Rmux pane clear/restart capability，因此 `project_clear` 与 `project_restart` 在 Rmux namespace 下应显式返回 unsupported，而不是构造 `TmuxBackend` 或拼接 Rmux CLI。

方案深度判断：clear/restart 属于长期维护的用户可见 runtime 操作，不能用字符串命令或假 pane 模拟补齐。当前选择显式 unsupported 是安全降级，因为它避免误操作 tmux authority；转正条件是后续为 Rmux 设计并验收 provider/runtime pane operation capability。

## Implementation Approach

- 复用 canonical namespace projection，继续让跨切读者从 namespace backend/ref 判断能力边界。
- 对 `project_focus`、`layout_status`、`project_clear`、`project_restart` 的 Rmux 路径采用明确状态输出，保留 tmux 路径既有行为。
- 为 unsupported 行为补回归测试，断言不会构造 `TmuxBackend`。

## Changes This Iteration

- `lib/ccbd/project_focus/tmux.py`：`backend_for_namespace()` 复用 `build_backend_for_namespace()`，让 Rmux 使用 canonical ref。
- `lib/cli/services/layout_status.py`：namespace payload 增加 canonical backend/ipc 字段；非 tmux backend 的 observation 返回 `skipped / unsupported_for_backend`。
- `lib/ccbd/handlers/project_clear.py`：非 tmux namespace 返回每个 agent 的 `skipped / unsupported_for_backend`，不再依赖 tmux socket。
- `lib/ccbd/handlers/project_restart.py`：`restart_project_agent_panes_in_place()` 对非 tmux namespace 返回 `skipped / unsupported_for_backend`。
- `test/test_ccbd_project_clear.py`：新增 Rmux namespace 测试，断言 `TmuxBackend` 不会被构造。
- `test/test_ccb_restart.py`：新增 Rmux namespace restart 测试，断言 `TmuxBackend` 不会被构造。
