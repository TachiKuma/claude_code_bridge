# Iteration 001

## Current Understanding

本 goal 的边界是把 Windows shell/log 语义收敛到统一 builder 边界，避免业务层继续拼 `sh -lc`、`tee -a`、PowerShell/cmd 字符串。

## Implementation Approach

采用边界抽取而不是 caller 分支化：新增统一 builder，保留旧 tmux helper 兼容包装，再逐步把 respawn、pipe-pane、clipboard policy 的调用层切到注入式消费。

## Changes This Iteration

- 新增 `lib/terminal_runtime/windows_shell_log_builder.py`，提供 `ShellResolution`、`resolve_shell()`、`resolve_shell_flags()`、`build_shell_command()`、`build_pipe_log_command()`、`append_stderr_redirection()`、`build_respawn_tmux_args()` 和 `clipboard_pipe_command()`。
- `lib/terminal_runtime/tmux_respawn.py` 改为兼容 facade，转调 builder。
- `lib/terminal_runtime/tmux_respawn_service.py` 接入 shell resolution 和 stderr redirection 的统一来源。
- `lib/terminal_runtime/tmux_logs.py` 的 `pipe-pane` 命令改为注入式 builder。
- `lib/terminal_runtime/tmux_backend_runtime/services.py` 开始统一装配 respawn/log builder。
- `lib/terminal_runtime/tmux_server_policy.py` 的 clipboard pipe command 改为从 builder 获取。
- 测试层把 clipboard helper 收口到统一 builder，并把相关用例名补成可被 `-k clipboard` 选中的回归入口。
