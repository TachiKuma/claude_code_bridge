# Iteration 001

## Current Understanding

本 goal 目标是完成 `rmux-send-capture-logging`：在已验收的 `RmuxBackend` core 上实现 pane IO 与 logging surface，并用 completion fixtures 证明 capture/log 文本形态不破坏 provider completion 判断。

边界保持不变：不接 ccbd lifecycle、foreground attach、mobile gateway production path，不修改 provider parser，不 fallback 到 tmux。

## Implementation Approach

采用 production Rmux IO module，而不是浅层 tmux adapter。

- `rmux_backend_runtime/io.py` 承载 send/key/capture/logging 编排。
- `RmuxBackend` 只公开 surface 和注入 `WindowsCommandBuilder`。
- `RmuxSubprocessCommandClient` 对 `capture-pane` 使用 binary capture，保留真实 stdout bytes。
- 测试使用 fake Rmux client 和 completion fixtures，避免依赖真实 daemon。
