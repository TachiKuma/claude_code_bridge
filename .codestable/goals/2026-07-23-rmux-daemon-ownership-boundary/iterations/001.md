# Iteration 001

## Current Understanding

本 goal 目标是完成 `rmux-daemon-ownership-boundary`：定义 Rmux daemon discovery、start、health、crash、cleanup 的 ownership 边界和 diagnostics evidence。

关键边界是：Rmux daemon 只能是 backend evidence，不能成为 ccbd lease holder、keeper lifecycle owner 或 provider kill eligibility。

## Implementation Approach

采用 contract/evidence 层实现，不实现 Rmux backend core、IPC 或真实 daemon lifecycle。

- 在 `terminal_runtime` 新增 Rmux daemon evidence contract。
- 用 deterministic helper 表达 start_result、cleanup plan 和 diagnostics projection。
- 在 namespace summary 投影 `backend_daemon_*`，但只在真实 rmux evidence 存在时投影，避免 tmux path 漂移。
- 用 focused tests 覆盖 authority boundary、cleanup 默认安全策略、diagnostics no-clobber 和 provider health evidence。

## Changes This Iteration

- 新增 `lib/terminal_runtime/rmux_daemon_contract.py`。
