```python
def t(key: str, **kwargs) -> str:
    """Get translated message by key.

    Args:
        key: Message key
        **kwargs: Format arguments

    Returns:
        Translated and formatted message
    """
    lang = get_lang()
    messages = MESSAGES.get(lang, MESSAGES["en"])
    msg = messages.get(key)
    if msg is None:
        msg = MESSAGES["en"].get(key, key)
    if kwargs:
        try:
            msg = msg.format(**kwargs)
        except (KeyError, ValueError):
            pass
    return msg

def detect_language() -> str:
    """Detect language from environment.

    Priority:
    1. CCB_LANG environment variable (zh/en/auto)
    2. System locale
    3. Default to English
    """
    ccb_lang = os.environ.get("CCB_LANG", "auto").lower()
    if ccb_lang in ("zh", "cn", "chinese"):
        return "zh"
    if ccb_lang in ("en", "english"):
        return "en"
    # Auto-detect from system locale
    try:
        lang = os.environ.get("LANG", "") or os.environ.get("LC_ALL", "")
        if not lang:
            lang, _ = locale.getdefaultlocale()
            lang = lang or ""
        lang = lang.lower()
        if lang.startswith("zh") or "chinese" in lang:
            return "zh"
    except Exception:
        pass
    return "en"
```

**1. 模块概述**
- 目标：为 CCB 提供可扩展的国际化框架
- 范围：仅 CCB（GSD 多语言功能冻结，per D-18）
- 核心能力：命名空间隔离、外部翻译覆盖、高性能查找

**2. 命名空间设计（per D-01）**
- 键格式：`{namespace}.{category}.{specific}`
- 示例：`ccb.error.no_terminal`, `ccb.startup.backend_started`, `common.error.file_not_found`
- 命名空间列表：
  - `ccb.*` - CCB 特定消息
  - `common.*` - 共享消息（预留，当前仅 CCB 使用）
- 冲突避免：不同命名空间的键完全隔离

**3. 核心类设计：I18nCore**
```python
class I18nCore:
    def __init__(self, namespace: str = "ccb")
    def load_translations(self) -> None  # 启动时一次性加载（per D-05）
    def _detect_language(self) -> str    # 语言检测（per D-06）
    def t(self, key: str, **kwargs) -> str  # 翻译函数（per D-03）
```

**4. 翻译键回退机制（per D-02）**
- 查找顺序：
  1. 外部翻译文件（~/.ccb/i18n/{namespace}/{lang}.json）
  2. 内置翻译文件（lib/i18n/{namespace}/{lang}.json）
  3. 英文回退（如果当前语言非英文）
  4. 返回键名本身（便于调试）
- 示例：t("ccb.unknown.key") 返回 "ccb.unknown.key"

**5. 语言检测逻辑（per D-06）**
- 优先级：
  1. CCB_LANG 环境变量（zh/cn/chinese → zh, en/english → en）
  2. 系统 locale（LANG, LC_ALL, LC_MESSAGES）
  3. 默认英文
- 实现细节：使用 locale.getdefaultlocale() 解析复杂格式（zh_CN.UTF-8）

**6. 外部翻译目录支持（per D-04, D-17）**
- 目录位置：~/.ccb/i18n/{namespace}/{lang}.json
- 加载优先级：外部翻译覆盖内置翻译（使用 dict.update()）
- 错误处理：目录/文件不存在时优雅降级，不中断启动
- 权限问题：记录警告但继续运行

**7. 性能优化（per D-05）**
- 启动时一次性加载所有翻译到内存（self.translations: Dict[str, str]）
- 查找时间：O(1) 字典查找
- 内存占用：预估 ~5KB（基于 Phase 1 分析：56 条消息 ≈ 1.8KB）
- 无需按需加载（当前翻译量小）

**8. API 兼容性**
- 保持现有 t(key, **kwargs) 签名（per D-03）
- 参数格式化：使用 str.format(**kwargs)
- 错误处理：格式化失败时返回未格式化消息（不抛出异常）

**9. 与现有 lib/i18n.py 的关系**
- 迁移路径：lib/i18n.py 将被 i18n_core 替代
- 兼容性：保持 t() API 完全兼容，确保平滑迁移
- 差异：新增命名空间、外部文件支持

```json
{
```

**3. 文件命名约定**
- 语言代码：ISO 639-1 两字母代码（en, zh）

**4. 加载优先级（per D-04, D-17）**

3. lib/i18n/ccb/en.json（英文回退）
4. 返回键名 "ccb.error.no_terminal"（最终回退）

**5. 外部翻译覆盖机制**
- 用户可在 ~/.ccb/i18n/ 创建同名文件
- 仅需覆盖特定键，无需复制完整文件
- 示例：用户仅覆盖 1 个键，其余使用内置翻译
```json
// ~/.ccb/i18n/ccb/zh.json（用户自定义）

}
```

**6. 从现有 lib/i18n.py 迁移**
- 现有结构：Python 字典 MESSAGES["zh"]["key"]

  1. 提取 MESSAGES["zh"] 字典
  2. 为每个键添加 "ccb." 前缀
  3. 写入 lib/i18n/ccb/zh.json
  4. 重复英文翻译

**7. 命名空间到目录的映射**
- ccb.* 键 → lib/i18n/ccb/{lang}.json
- common.* 键 → lib/i18n/common/{lang}.json
- 规则：键的第一部分（点号前）对应目录名

**8. 文件大小估算**
- 基于 Phase 1 分析：56 条消息 ≈ 1.8KB
- 预估 CCB 翻译文件：~3-5KB/语言
- 外部翻译文件：通常 <1KB（仅覆盖少量键）

**9. 错误处理**
- 文件不存在：跳过，使用回退机制
- JSON 解析失败：记录警告，跳过该文件
- 权限错误：记录警告，跳过外部翻译
- 不中断启动：所有错误都优雅降级

**10. 扩展性考虑**

- 新增命名空间：创建新目录（如 gsd/）
