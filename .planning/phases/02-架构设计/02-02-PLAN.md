**1. TaskHandle 数据结构（per D-09）**
```python
@dataclass
class TaskHandle:
    """任务句柄，用于跟踪提交的任务"""
    task_id: str          # 唯一任务标识符，格式：{provider}_{timestamp_ms}
    provider: str         # AI 提供商名称（codex, droid, gemini, claude）
    timestamp: float      # 提交时间戳（Unix 时间，秒）
```

字段说明：
- task_id: 唯一标识符，格式 "{provider}_{int(time.time() * 1000)}"，示例 "codex_1711234567890"
- provider: 提供商名称，必须是 ["codex", "droid", "gemini", "claude"] 之一
- timestamp: 提交时间，使用 time.time() 获取，用于超时检测和日志记录

**2. TaskResult 数据结构（per D-09, D-10）**
```python
@dataclass
class TaskResult:
    """任务结果，包含执行状态和输出"""
    task_id: str                    # 对应的任务 ID
    status: str                     # 状态：pending, completed, error
    output: Optional[str] = None    # 成功时的输出内容
    error: Optional[str] = None     # 失败时的错误信息
```

字段说明：
- task_id: 与 TaskHandle.task_id 对应
- status: 三种状态
  - "pending": 任务仍在处理中，output 和 error 均为 None
  - "completed": 任务成功完成，output 包含结果，error 为 None
  - "error": 任务失败，error 包含错误信息，output 为 None
- output: AI 的回复内容（仅 status="completed" 时有值）
- error: 错误描述（仅 status="error" 时有值）

**3. 状态转换图**
```
[submit] → TaskHandle
           ↓
[poll]   → TaskResult(status="pending")
           ↓
[poll]   → TaskResult(status="completed", output="...")
           或
[poll]   → TaskResult(status="error", error="...")
```

**4. 错误处理策略（per D-10）**
- 不抛出异常：所有错误通过 TaskResult(status="error") 返回
- 一致性接口：poll() 始终返回 TaskResult，调用者检查 status 字段
- 错误类型：
  - 提供商不可用：error="Provider {provider} not available"
  - 超时：error="Timeout waiting for {provider} reply"
  - 命令执行失败：error="Command failed: {stderr}"

**5. 类型注解**
```python
from dataclasses import dataclass
from typing import Optional

@dataclass
class TaskHandle:
    task_id: str
    provider: str
    timestamp: float

@dataclass
class TaskResult:
    task_id: str
    status: str
    output: Optional[str] = None
    error: Optional[str] = None
```

**6. 使用示例**
```python
# 提交任务
handle = backend.submit("codex", "Review this code")
# TaskHandle(task_id="codex_1711234567890", provider="codex", timestamp=1711234567.89)

# 轮询结果（pending）
result = backend.poll(handle)
# TaskResult(task_id="codex_1711234567890", status="pending", output=None, error=None)

# 轮询结果（completed）
result = backend.poll(handle)
# TaskResult(task_id="codex_1711234567890", status="completed", output="Code looks good", error=None)

# 轮询结果（error）
result = backend.poll(handle)
# TaskResult(task_id="codex_1711234567890", status="error", output=None, error="Provider timeout")
```

**7. 与现有 CCB 命令的映射**
- TaskHandle.provider → ask 命令的第一个参数
- TaskResult.output → pend 命令的 stdout
- TaskResult.error → pend 命令的 stderr 或返回码非 0

**8. 序列化支持**
- dataclass 自动支持 asdict() 转换为字典
- 可用于 JSON 序列化：json.dumps(asdict(handle))
- 可用于日志记录和调试

**1. 接口概述**
- 目标：为 GSD 提供标准化的多 AI 协作接口

class CCBCLIBackend:
    """CCB CLI 包装接口，提供结构化的多 AI 协作能力"""

    def submit(self, provider: str, prompt: str, context: Optional[Dict] = None) -> TaskHandle
    def poll(self, handle: TaskHandle) -> TaskResult
    def ping(self, provider: str) -> bool

**3. submit() 方法设计（per D-08）**
```python
def submit(self, provider: str, prompt: str, context: Optional[Dict] = None) -> TaskHandle:
    """提交任务到 AI 提供商

    Args:
        provider: 提供商名称（codex, droid, gemini, claude）
        prompt: 提示词内容
        context: 可选的上下文信息（预留，当前未使用）

    Returns:
        TaskHandle: 任务句柄，用于后续轮询

    Implementation:
        1. 验证 provider 在可用列表中
        2. 构造命令：["ask", provider, prompt]
        3. 使用 subprocess.run() 执行（后台模式，添加 & 符号）
        4. 生成 task_id: f"{provider}_{int(time.time() * 1000)}"
        5. 返回 TaskHandle(task_id, provider, time.time())
    """
```

实现细节：
- 命令格式：`ask {provider} "{prompt}" &`（后台执行）
- 不等待命令完成，立即返回 TaskHandle
- 错误处理：provider 无效时返回 TaskHandle，poll() 时返回 error

**4. poll() 方法设计（per D-08）**
```python
def poll(self, handle: TaskHandle) -> TaskResult:
    """轮询任务结果

    Args:
        handle: submit() 返回的任务句柄

    Returns:
        TaskResult: 任务结果（pending/completed/error）

    Implementation:
        1. 构造命令：[f"{handle.provider[0]}pend"]（如 cpend, dpend）
        2. 使用 subprocess.run(capture_output=True, text=True) 执行
        3. 检查返回码：
           - returncode != 0 → TaskResult(status="error", error=stderr)
           - stdout 非空 → TaskResult(status="completed", output=stdout.strip())
           - stdout 为空 → TaskResult(status="pending")
    """
```

实现细节：
- 命令映射：codex → cpend, droid → dpend, gemini → gpend
- 超时处理：使用 subprocess timeout 参数（默认 5 秒）
- 输出解析：直接使用 stdout，不解析特定格式

**5. ping() 方法设计（per D-08）**
```python
def ping(self, provider: str) -> bool:
    """检查提供商连接状态

    Args:
        provider: 提供商名称

    Returns:
        bool: True 表示可用，False 表示不可用

    Implementation:
        1. 构造命令：[f"{provider[0]}ping"]（如 cping, dping）
        2. 使用 subprocess.run() 执行
        3. 返回 returncode == 0
    """
```

实现细节：
- 命令映射：codex → cping, droid → dping, gemini → gping
- 超时：2 秒（快速检测）
- 不捕获输出，仅检查返回码

**6. list_providers() 方法设计（per D-08）**
```python
def list_providers(self) -> List[str]:
    """列出所有可用的 AI 提供商

    Returns:
        List[str]: 提供商名称列表

    Implementation:
        1. 执行命令：["ccb-mounted"]
        2. 解析 stdout，按行分割
        3. 返回提供商列表（如 ["codex", "droid", "gemini"]）
    """
```

实现细节：
- 命令：ccb-mounted（CCB 内置命令）
- 输出格式：每行一个提供商名称
- 错误处理：命令失败时返回空列表 []

**7. 错误处理策略（per D-10）**
- submit(): 始终返回 TaskHandle，错误延迟到 poll() 时报告
- poll(): 返回 TaskResult(status="error")，不抛出异常
- ping(): 返回 False，不抛出异常
- list_providers(): 返回空列表 []，不抛出异常

**8. subprocess 包装模式（per D-11）**
```python
import subprocess
from typing import List, Optional

# 提交任务（后台）
result = subprocess.run(
    ["ask", provider, prompt],
    capture_output=False,  # 不捕获输出（后台运行）
    text=True
)

# 轮询结果
result = subprocess.run(
    [f"{provider[0]}pend"],
    capture_output=True,
    text=True,
    timeout=5
)

# 检查连接
result = subprocess.run(
    [f"{provider[0]}ping"],
    capture_output=False,
    timeout=2
)
```

**9. 与 CCB 命令的映射表**
| 方法 | CCB 命令 | 说明 |
|------|----------|------|
| submit("codex", "...") | ask codex "..." & | 后台提交 |
| poll(handle) | cpend | 轮询 Codex 结果 |
| poll(handle) | dpend | 轮询 Droid 结果 |
| poll(handle) | gpend | 轮询 Gemini 结果 |
| ping("codex") | cping | 检查 Codex 连接 |
| list_providers() | ccb-mounted | 列出已挂载提供商 |

**10. 使用示例**
```python
backend = CCBCLIBackend()

# 检查可用提供商
providers = backend.list_providers()
# ["codex", "droid", "gemini"]

# 提交任务
handle = backend.submit("codex", "Review this code")

# 轮询结果
while True:
    result = backend.poll(handle)
    if result.status == "completed":
        print(result.output)
        break
    elif result.status == "error":
        print(f"Error: {result.error}")
        break
    time.sleep(1)
```

**11. 线程安全性**
- 当前设计：非线程安全（subprocess 调用无锁）
- 并发场景：多个 submit() 可并行调用（不同提供商）
- 限制：同一提供商的 poll() 应串行调用（避免竞态条件）

**12. 未来扩展**
- context 参数：预留用于传递文件路径、项目信息等
- 超时配置：可添加 timeout 参数到 poll()
- 批量操作：可添加 submit_batch() 方法
