From lib/compat.py:
```python
def setup_windows_encoding() -> None: ...
def decode_stdin_bytes(data: bytes) -> str: ...
def read_stdin_text() -> str: ...
```

Encoding fallback chain in decode_stdin_bytes:
1. BOM detection (UTF-8 BOM, UTF-16 LE BOM, UTF-16 BE BOM)
2. CCB_STDIN_ENCODING env override
3. UTF-8 strict
4. locale.getpreferredencoding() strict
5. mbcs strict (Windows only)
6. UTF-8 with errors="replace" (last resort)

From ccb (main entry):
- Path normalization uses: os.path.normpath(), Path.resolve(), Windows/WSL/MSYS path conversion
- WSL detection: sys.platform == "linux" and os.path.exists("/proc/version")
- MSYS detection: check for MSYS_PREFIX env or /usr/bin in path
</interfaces>

    Import pattern:

    Key implementation notes for each test:

    **test_decode_gbk_fallback:**
    - Encode "中文测试" with `"中文测试".encode("gbk")`
    - Mock `locale.getpreferredencoding` to return `"gbk"` using `mock.patch("locale.getpreferredencoding", return_value="gbk")`
    - Mock `sys.platform` to `"win32"` (needed for mbcs path to be considered)
    - Assert `"中文"` in result

    **test_decode_windows_1252_fallback:**
    - Create em dash string: `"\u2014em dash\u2014"`
    - Encode with `.encode("windows-1252")`
    - Mock locale to return `"cp1252"`, sys.platform to `"win32"`
    - Assert `"em dash"` in result

    **test_decode_shift_jis_no_crash:**
    - Encode `"テスト".encode("shift_jis")`
    - Mock locale to return `"utf-8"` (so shift_jis doesn't match), sys.platform to `"win32"`
    - Assert `isinstance(result, str)` — key: must not crash

    **test_decode_mbcs_windows_data_loss:**
    - This is the pitfall from RESEARCH.md: mbcs maps to system ANSI code page, not Shift-JIS
    - On win32, when locale is "utf-8" and mbcs maps to cp1252/cp936, Shift-JIS input will fall through to final `utf-8 with replace`
    - Assert result is str (not bytes, not exception)

    **test_decode_env_override:**
    - Set `os.environ["CCB_STDIN_ENCODING"] = "utf-8"` via monkeypatch
    - Encode with GBK: `"中文测试".encode("gbk")`
    - Mock locale to return `"gbk"`
    - Since env override forces UTF-8, GBK bytes will fail strict UTF-8, then use errors="replace"
    - Assert result is str (may have replacement chars)

    **test_decode_mixed_surrogate_free:**
    - Create bytes that could produce surrogates: e.g., random bytes or specific encoding mismatches
    - Call decode_stdin_bytes
    - Assert no characters in range U+D800-U+DFFF in result: `all(ord(c) < 0xD800 or ord(c) > 0xDFFF for c in result)`

    All tests: `@pytest.mark.windows`, `@pytest.mark.compat`.
