    **1. Fix: WSL probe timeout on native Windows (Critical/High — lib/ccb_config.py)**
    - Add a fast pre-check at the start of `_wsl_probe_distro_and_home()`:
    ```python
    def _wsl_probe_distro_and_home() -> tuple[str, str]:
        # Fast pre-check: is WSL available at all?
        try:
            r = subprocess.run(
                ["wsl.exe", "--list", "-q"],
                capture_output=True, text=True, encoding="utf-16-le", errors="replace", timeout=2,
                **_subprocess_kwargs()
            )
            if r.returncode != 0:
                # WSL not installed or not functional — return defaults immediately
                return "Ubuntu", "/root"
        except (subprocess.TimeoutExpired, FileNotFoundError, OSError):
            return "Ubuntu", "/root"
        # ... existing probe logic (with reduced timeout)
    ```
    - Also reduce the existing timeout from 10s to 5s for the main probe call
    - Rationale: Fast pre-check (< 2s) catches "no WSL" case. Main probe still works when WSL is installed.

    **2. Fix: mbcs encoding fallback data loss (High — lib/compat.py:61-65)**
    - Change `data.decode("mbcs", errors="strict")` to `data.decode("mbcs", errors="replace")`
    - Rationale: mbcs strict either succeeds with possibly wrong decoding or fails. Using `errors="replace"` ensures we never crash and always produce valid string output.
    - The final fallback already uses `errors="replace"`, so mbcs with replace is consistent.

    **3. Fix: bin/ask PowerShell path escaping incomplete (High — bin/ask:675-677)**
    - Create a helper function at the top of the Windows section:
    ```python
    def _escape_ps_path(path_str: str) -> str:
        """Escape PowerShell special characters in file path for embedding in PS script."""
        s = str(path_str)
        s = s.replace("'", "''")     # single-quote escape for single-quoted strings
        s = s.replace('`', '``')     # backtick escape
        return s
    ```
    - Then use single-quoted strings in the PowerShell script (single quotes prevent `$` expansion):
    ```python
    status_file_win = _escape_ps_path(status_file)
    log_file_win = _escape_ps_path(log_file)
    # In the PS script, use single quotes: $statusFile = '{status_file_win}'
    ```
    - Verify all path references in the generated PS script use single quotes
