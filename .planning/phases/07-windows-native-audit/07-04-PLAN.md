    **Known fixes to pre-populate (from RESEARCH.md analysis):**

    **WIN-02-001: mbcs encoding fallback data loss**
    - Fix: Replace `data.decode("mbcs", errors="strict")` with `data.decode("utf-8", errors="replace")` in the Windows fallback path
    - Or: Add `errors="replace"` to mbcs call instead of `errors="strict"`
    - File: lib/compat.py:61-65
    - Why: mbcs strict mode either succeeds (possibly wrong decoding) or fails. With replace, we get safe output.

    **WIN-02-002: WSL probe 10s timeout on native Windows**
    - Fix: Add `wsl.exe --list -q` fast check (< 2s timeout) before calling `_wsl_probe_distro_and_home`. If fast check fails, return default values immediately.
    - File: lib/ccb_config.py:128-168 (apply_backend_env function)
    - Why: Saves 10s on every daemon startup when WSL is not installed.

    **WIN-03-001: os.chmod(0o600) silent on NTFS in mail/ modules**
    - Fix: Add `if os.name != "nt":` guard before each chmod call in lib/mail/daemon.py and lib/mail/credentials.py
    - File: lib/mail/daemon.py:96, lib/mail/credentials.py:113, and other mail/ chmod locations
    - Why: Matches the pattern already used in lib/askd_server.py:297

    **WIN-02-003: bin/ask PowerShell path escaping incomplete**
    - Fix: Expand escaping to handle `$`, backtick, and single-quote characters in paths
    - File: bin/ask:675-677
    - Current: `str(path).replace('"', '`"')`
    - Fixed: Also replace `$` → `` `$ ``, backtick → `` `` ``, single-quote → `''`

    **WIN-01-001: daemon cold start exceeds 3s (if test fails)**
    - Fix: Depends on measurement — likely caused by WSL probe timeout (WIN-02-002). Fix WSL probe first.
    - File: lib/ccb_config.py:171-188 (apply_backend_env)

    Order fixes by dependency: WSL probe timeout fix first (affects performance), then other fixes.

    2. Current code snippet
    3. Proposed fixed code snippet
    4. Why this approach (alternatives considered)
    5. Which test verifies the fix
    6. Risk assessment (could the fix break existing functionality?)
