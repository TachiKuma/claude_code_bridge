    - Test 2: `test_command_response_under_500ms` — if daemon is running from test 1, sends a ping via socket (read state file for port/token), measures round-trip, asserts < 0.5s. Per D-07. If daemon not running, start fresh.
    - Test 3: `test_daemon_memory_under_50mb` — if psutil is available, reads daemon process RSS after startup, asserts < 50*1024*1024 bytes. Per D-08. If psutil not available, pytest.mark.skip with reason.
    - Test 4: `test_python_module_import_time` — measures `timeit.timeit('import askd_server', setup='import sys; sys.path.insert(0, "lib")')`, reports duration (informational, no hard assert — baseline for future regression).

  <action>
    Create `tests/windows/test_perf_daemon.py` with the following structure:

    All tests use `pytest.mark.windows` and `pytest.mark.perf` markers.

    **test_daemon_cold_start_under_3s:**
    - Record `start = time.perf_counter()`
    - Start subprocess: `subprocess.Popen([sys.executable, "bin/askd"], stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL, creationflags=CREATE_NO_WINDOW if os.name=="nt" else 0)`
    - Poll for state file at `run_dir() / "askd.json"` every 100ms, up to 5s timeout
    - `elapsed = time.perf_counter() - start`
    - `assert elapsed < 3.0, f"daemon cold start took {elapsed:.2f}s (limit: 3s)"`
    - Store daemon proc and port in module-level variable for subsequent tests
    - Cleanup: `proc.terminate()` in fixture or teardown

    **test_command_response_under_500ms:**
    - Read state file to get port and token
    - Connect to `127.0.0.1:port` via `socket.create_connection()`, set 2s timeout
    - Send: `json.dumps({"type": "askd.ping", "v": 1, "id": "perf-test", "token": token}) + "\n"`
    - Read response line
    - `assert elapsed < 0.5, f"ping round-trip took {elapsed:.3f}s (limit: 500ms)"`

    **test_daemon_memory_under_50mb:**
    - Try `import psutil`
    - If ImportError: `pytest.skip("psutil not installed")`
    - Get daemon PID from state file
    - `rss = psutil.Process(pid).memory_info().rss`
    - `assert rss < 50 * 1024 * 1024, f"daemon RSS {rss / 1024 / 1024:.1f}MB (limit: 50MB)"`

    **test_python_module_import_time:**
    - Use `timeit.timeit(stmt='import askd_server', setup='import sys; sys.path.insert(0, "lib")', number=3)`
    - Log result with `pytest.mark.xfail(reason="informational only")` or just print to stdout
    - No assertion — this is a baseline measurement

    Important: All tests that interact with the real daemon must use `temp_run_dir` fixture to isolate from any existing daemon state. Use `isolated_env` fixture to prevent CCB_* env var leakage.

    The daemon startup subprocess should capture CCB_RUN_DIR env var pointing to temp directory. Add `env={**os.environ, "CCB_RUN_DIR": str(temp_dir)}` to Popen call.
