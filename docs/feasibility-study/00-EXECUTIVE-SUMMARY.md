# GSD & CCB 国际化与多 AI 协作可行性研究 -- 执行摘要

**项目全称:** GSD & CCB 国际化与多 AI 协作可行性研究
**研究日期:** 2026-03-28 至 2026-03-30
**文档性质:** 面向管理层和快速阅读者的统一入口

---

## 项目概述

本项目是一项为期 3 天的可行性研究，旨在评估为 GSD (Get Shit Done) 和 CCB (Claude Code Bridge) 进行国际化 (i18n) 改造并实现多 AI 协作能力的技术可行性。

**核心价值:** 通过国际化和多 AI 协作，让 GSD 和 CCB 能够服务更广泛的用户群体，并显著提升复杂任务的执行质量。

**研究范围:**
- 语言支持聚焦中英文，其他语言预留扩展接口
- 基于现有 Python 技术栈，不引入重型框架
- 不破坏现有功能和 API 兼容性
- 不追求生产级完整实施，以技术验证和方案设计为主

---

## 研究方法

本研究采用 5 阶段递进方法，每个阶段产出明确的分析或设计文档：

1. **代码库分析** -- 识别所有需要国际化的文本位置，区分人类可读文本和协议字符串
2. **架构设计** -- 设计共享 i18n_core 模块和多 AI 协作接口
3. **风险评估** -- 评估实施风险、工作量和缓解策略
4. **原型验证** -- 用最小实现验证关键技术点
5. **文档交付** -- 综合全部成果形成结论性文档（当前阶段）

**关键数据基线:**
- CCB: 98 个 Python 文件，约 6,366 行代码
- GSD: 18 个 JavaScript 文件
- 识别的人类可读文本总量: 9029 条
- 协议字符串白名单: 300 项（7 个分类）
- 原型验证单元测试: 57 个，全部通过

---

## 核心结论

本研究围绕四个技术方向进行了深入分析和验证，结论如下：

| 方向 | 结论 | 关键数据 | 建议 |
|------|------|----------|------|
| i18n 架构 | 可行，I18nCore 原型验证通过 | 56 条消息迁移，11 个测试通过 | 推荐按 536h 实施或 26h 原型先验证 |
| 协议保护 | 双层机制验证通过 | 300 项白名单，CI + 运行时双层 | P0 必须实现 |
| 多 AI 集成 | 可行，CCBCLIBackend 原型通过 | 25 个 mock 测试通过，退出码映射正确 | 推荐按 40-60h 实施 |
| 文件锁 | 跨平台验证通过 | Windows/Unix 双平台，9 个测试通过 | 可直接复用 |

**综合判断:** 国际化改造和多 AI 集成在技术层面均可行。关键风险（协议字符串误翻译、多 AI 并发冲突）已有对应的设计缓解方案并经过原型验证。建议先以 26 小时原型验证验证生产环境适配性，再决定是否推进 536 小时的完整实施。

## 风险概要

本研究的风险评估阶段识别了以下关键风险及对应的缓解措施：

**Critical 级别:**
- 协议字符串误翻译 -- 协议字符串（如命令名、环境变量、完成标记）被错误翻译会导致跨进程通信失败
  - 缓解: 双层保护机制（CI 检查阻止合并 + 运行时验证记录警告），300 项白名单覆盖 7 个分类
  - 状态: 原型验证通过，12 个测试确认保护有效

**High 级别:**
- 多 AI 单 provider 并发 -- 同一 AI 提供商同时提交多个任务会互相覆盖
  - 缓解: 单任务约束（每个 provider 同时仅一个活跃任务），ProviderLock 文件锁串行化
  - 状态: CCBCLIBackend 原型已实现退出码映射和 ProviderLock 集成

**Medium 级别:**
- 文件锁超时 -- 锁获取超时可能导致误报或体验下降
  - 缓解: 可配置超时参数，过期锁自动检测（PID 存活检查）
  - 状态: FileLock 原型验证通过，跨平台兼容 Windows/Unix
- 跨平台编码差异 -- Windows 控制台编码可能影响中文显示
  - 缓解: 统一使用 UTF-8 编码，subprocess 添加隐藏窗口标志
  - 状态: CCBCLIBackend 已添加 Windows 兼容代码

**缓冲建议:**
- i18n 完整实施建议预留 +20% 缓冲（536h 基线 -> 643h 排期口径）
- 多 AI 集成建议预留缓冲后口径 60h（中位估算 52h）

## 工作量总览

| 工作项 | 工时（基线） | 工时（缓冲口径） | 时间跨度 |
|--------|-------------|-----------------|---------|
| i18n 完整实施 | 536 小时 | 643 小时 | 约 8 周（1 开发 + 1 翻译并行） |
| 多 AI 集成 | 52 小时 | 60 小时 | 约 1.5 周（1 开发） |
| i18n 原型验证 | 26 小时 | 30 小时 | 约 3-4 天（已完成） |

**资源配置建议:**
- i18n 完整实施: 1 名 Python/CLI 开发者 + 1 名翻译/审校人员，并行推进约 8 周
- 多 AI 集成: 1 名 Python 开发者，约 6.5 个工作日
- 两项工作不互为强依赖，可独立推进

**已完成的原型验证工作:**
- I18nCore -- 56 条消息迁移至 en/zh/xx.json，11 个测试通过，向后兼容层验证
- CCBCLIBackend -- 25 个 mock 测试覆盖 submit/poll/ping/list_providers 及错误路径
- 协议保护 -- CI 脚本加载 300 项白名单，干净翻译 PASS，违规翻译 FAIL（检测 3 个违规项）
- FileLock -- 通用跨平台文件锁，9 个测试通过，支持 Windows (msvcrt) 和 Unix (fcntl)
